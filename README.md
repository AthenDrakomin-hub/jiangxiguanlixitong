# 江西酒店管理系统

现代化的酒店管理系统，集成了客房管理、点餐服务、财务管理和用户权限控制等功能。

## 功能特性

- **酒店客房管理**：支持多楼层房间管理，包括标准间、VIP包间等不同类型
- **客户点餐服务**：客人可通过二维码扫码点餐，支持多种支付方式
- **订单管理**：完整的订单生命周期管理（待接单、制作中、已送达、已完成）
- **财务管理**：支出记录、营收统计、财务报表
- **库存管理**：食材和用品库存跟踪，低库存提醒
- **用户权限管理**：多角色权限控制（管理员、经理、员工）
- **云数据同步**：支持云端数据备份和同步
- **多语言支持**：支持中文和英文界面

## 系统架构

### 技术栈
- **前端**: React 18 + TypeScript + Tailwind CSS
- **后端**: Vercel Edge Functions
- **数据库**: 支持多种后端（内存数据库用于开发，Neon PostgreSQL 用于生产）
- **UI库**: Lucide React 图标，Tailwind CSS 样式
- **状态管理**: React Hooks + 自定义 hooks

### 数据库支持
- 内存数据库（开发环境）
- Neon PostgreSQL（生产环境）
- 自动数据验证和序列化

### 核心模块
- **酒店系统**: 房间管理、客房点餐
- **菜单管理**: 菜品分类、价格管理
- **订单管理**: 订单流程控制
- **财务管理**: 支出记录、营收统计
- **库存管理**: 库存跟踪、采购管理
- **用户管理**: 权限控制、角色管理

## 环境配置

### 开发环境
```bash
npm install
npm run dev
```

### 生产环境部署
```bash
# 部署到 Vercel
vercel deploy
```

### 环境变量
- `DB_TYPE`: 数据库类型 (memory/neon)
- `NEON_CONNECTION_STRING`: Neon 数据库连接字符串 (生产环境)
- `VITE_ADMIN_USER`: 管理员用户名
- `VITE_ADMIN_PASS`: 管理员密码

## 数据库初始化

系统支持一键数据初始化，包括：
- 64间酒店房间 (8201-8232, 8301-8332)
- 预设菜品和分类
- 支付方式配置
- 系统基础设置

## API 接口

系统提供完整的 RESTful API：
- `/api/hotel_rooms` - 酒店房间管理
- `/api/dishes` - 菜品管理
- `/api/orders` - 订单管理
- `/api/expenses` - 支出管理
- `/api/inventory` - 库存管理
- `/api/users` - 用户管理
- `/api/system_settings` - 系统设置

## 部署说明

1. **本地开发**：使用内存数据库，无需额外配置
2. **生产部署**：配置 Neon 数据库连接，启用数据持久化
3. **初始化数据**：部署后调用 `/api/seed` 接口初始化基础数据
